# Branding & packaging

The extension was renamed **Agent Kanban → Agentic Kanban**. This is a *visible* rename with *compatibility*: internal identifiers stay.

## What changes vs. what stays

**Change to `Agentic Kanban` (user-visible):**
- Display copy in `README.md`, `TECHNICAL.md`, root `AGENTS.md`, `src/webview/board.ts` (uninit panel), user-facing strings in `src/WorktreeService.ts` and `src/SlashCommandProvider.ts`.
- `package.json` user-facing text metadata (`displayName`, `description`, view/command/config titles, chat participant `fullName`).
- Managed sentinel + heading written by `src/agents/ChatParticipant.ts`.
- Runtime log filename: `agent-kanban.log` → `agentic-kanban.log` (`src/LogService.ts`).
- The `.agentkanban/.gitignore` header generated by `src/BoardConfigStore.ts` (`# Agentic Kanban - auto-generated`).

**Keep unchanged (compatibility):**
- Command IDs and config keys: `agentKanban.*`.
- Workspace directory: `.agentkanban/`.
- Git branch prefix: `agentkanban/`.
- Chat participant id: `agentKanban.chat`.
- Canonical repository, homepage, and issue URLs use `milzamsz/vscode-agentic-kanban`.
- Marketplace links use `milzam.agentic-kanban`; Open VSX links use `milzam/agentic-kanban`.
- Upstream repository links are retained only in the README Credits section.

## Sentinel compatibility rule

- New output: `<!-- BEGIN AGENTIC KANBAN — DO NOT EDIT THIS SECTION -->` … `<!-- END AGENTIC KANBAN -->` (note the em dash `—`, matched verbatim).
- Read/replace: recognise **both** the new markers and the legacy `<!-- BEGIN AGENT KANBAN … -->` / `<!-- END AGENT KANBAN -->`, upgrading an old section in place.
- The legacy constants live in `ChatParticipant.ts` as `AGENTS_MD_BEGIN_LEGACY` / `AGENTS_MD_END_LEGACY` and are intentionally never written.

## Release / packaging pipeline

Run all four, in order, before claiming packaging work is done:

```
npm run lint            # tsc --noEmit
npm test                # vitest run
npm run build           # node build.mjs (esbuild + tailwind)
npx @vscode/vsce package
```

Regenerate `.agentkanban/INSTRUCTION.md` from `assets/INSTRUCTION.md` only if they have drifted.

## Search validation

```
rg "Agent Kanban|AGENT KANBAN|agent-kanban\.log" src/ README.md TECHNICAL.md AGENTS.md
```
Allowed survivors only:
- Legacy-compat constants/comments in `ChatParticipant.ts`.
- Legacy *input* fixtures in `src/test/ChatParticipant.test.ts` (they assert old markers upgrade to new ones).
- The intentional legacy note in `TECHNICAL.md`.
- Historical task records under `.agentkanban/tasks/**` are excluded from the VSIX (`.vscodeignore`) and are left as-is.

## VSIX verification checklist

`.vsix` is a zip. Copy to `*.zip`, extract, then inspect:

```
rg "Agent Kanban|AGENT KANBAN|agent-kanban\.log" <extracted>/extension --glob '!*.map'
```
Expected, and ONLY these:
- `dist/extension.js` — `AGENTS_MD_BEGIN_LEGACY` / `AGENTS_MD_END_LEGACY` constants.
- `TECHNICAL.md` — the legacy-recognition note.

Anything else (a heading, a button label, the log filename, `## Agent Kanban`) is a regression — fix the source and repackage. Confirm the visible brand in `package.json`, `readme.md`, `AGENTS.md`, and the bundled `board.js` reads `Agentic Kanban`.
